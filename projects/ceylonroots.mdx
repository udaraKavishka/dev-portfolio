## Problem

CeylonRoots began as a university group project — a Spring Boot/MySQL travel-booking platform with a full team split across frontend, backend, and design. After that phase wrapped, I kept building on it alone, and the direction changed: instead of a generic booking app, I turned it into a content-driven Sri Lanka travel site — a public blog and destination guide backed by an admin system content editors could actually use, without needing to touch code to publish a post.

## Architecture / Approach

- **Next.js (App Router)** serves the public site — blog, destinations, travel packages, testimonials — with SEO-oriented routing and dynamic metadata.
- **A separate Flask admin app** (`backend-flask/`) handles content management: blog post authoring, destination/package editing, gallery and testimonial moderation, all through server-rendered admin templates rather than a bolted-on CMS.
- **Subdomain-based routing** in Next.js middleware separates the public site from the admin surface, so `admin.` traffic resolves to the Flask-managed views while the main domain stays the fast, content-focused frontend.
- **A shared database** backs both apps, with the Flask side owning migrations and the Next.js side reading published content.
- **CI enforces backend hygiene** — a pre-commit-based GitHub Actions job runs ruff (lint + format), pyright (type checking), and djlint (template linting) on every push to `backend-flask/`, replacing an earlier Prettier-based setup that didn't actually check Python.

## My Contribution

I rebuilt the project from its original team-based Spring Boot/MySQL form into the current architecture entirely solo: designing the Flask admin app's models, views, and Jinja templates for every content type (blog, destinations, packages, gallery, testimonials, submissions), wiring up the Next.js middleware for subdomain-based admin routing, and setting up the CI pipeline (ruff/pyright/djlint pre-commit) that replaced a redundant Prettier + npm-audit setup with checks that actually catch Python bugs. I also wrote the site's travel blog content and migrated the testimonials system to be frontend-managed rather than hardcoded.

## Outcome

What started as a team exercise in a Spring Boot CRUD app became a solo-run, content-managed travel site with over 450 commits — a dual-app architecture where a non-technical content editor can publish a blog post through the Flask admin without a deploy, while the Next.js frontend stays fast and SEO-focused. It's the project where I learned the most about maintaining a codebase alone after a team disbands: the CI and pre-commit hooks exist specifically because there's no second reviewer catching mistakes anymore.
